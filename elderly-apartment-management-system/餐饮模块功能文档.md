# 老年公寓管理系统 - 餐饮模块功能文档

## 文档概述

本文档详细描述了老年公寓管理系统中餐饮模块的用户端APP开发需求与设计规范，涵盖页面架构、功能模块、交互流程、视觉设计规范及数据展示格式等内容。

---

## 一、页面整体架构设计

### 1.1 系统架构

餐饮模块采用前后端分离架构：
- **前端**：Vue 3 + Element Plus 技术栈
- **后端**：Spring Boot + MyBatis Plus
- **数据库**：MySQL

### 1.2 模块组成

餐饮模块包含以下核心子模块：

| 模块名称 | 功能说明 | 对应页面 |
|---------|---------|---------|
| 菜品管理 | 菜品信息维护、分类管理 | menu.vue |
| 每周食谱 | 食谱计划制定与发布 | plan.vue |
| 老人订餐 | 订餐记录管理与处理 | order.vue |
| 送餐记录 | 送餐状态跟踪与管理 | delivery.vue |
| 餐饮配送 | 配送任务调度与执行 | distribution.vue |

## 二、功能模块详细设计

### 2.1 菜品管理模块

#### 2.1.1 功能概述
菜品管理模块用于维护系统中的菜品信息，包括菜品的增删改查、分类管理等功能。

#### 2.1.2 页面布局
```
┌─────────────────────────────────────────────────────────────┐
│  菜品管理                                          [新增菜品] │
├─────────────────────────────────────────────────────────────┤
│  菜品名称: [________]  菜品类型: [全部▼]  [查询] [重置]       │
├─────────────────────────────────────────────────────────────┤
│  编号 │ 菜品名称 │ 类型 │ 价格 │ 主要食材 │ 营养成分 │ 操作  │
│  1    │ 小米粥   │ 早餐 │ ¥3.00│ 小米、水 │ ...      │ 编辑删除│
│  ...  │ ...      │ ...  │ ...  │ ...      │ ...      │ ...   │
├─────────────────────────────────────────────────────────────┤
│  共 XX 条  [10▼] 条/页  < 1 2 3 ... 10 >                    │
└─────────────────────────────────────────────────────────────┘
```

#### 2.1.3 功能点说明

**1. 菜品列表展示**
- 分页展示菜品信息，默认每页10条
- 支持按菜品名称模糊搜索
- 支持按菜品类型筛选（早餐/午餐/晚餐/通用）
- 表格字段：编号、菜品名称、菜品类型（标签展示）、价格、主要食材、营养成分、适宜人群、状态（启用/禁用）、操作

**2. 新增/编辑菜品**
- 菜品名称：必填，最大100字符
- 菜品类型：必填，单选（早餐/午餐/晚餐/通用）
- 价格：必填，数字输入，保留2位小数
- 主要食材：选填，多行文本
- 营养成分：选填，多行文本
- 适宜人群：选填，多行文本，提示如"糖尿病患者、高血压患者等"
- 状态：单选（启用/禁用）
- 备注：选填，多行文本

**3. 删除菜品**
- 二次确认弹窗
- 删除前校验是否被食谱引用

#### 2.1.4 API接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| /meal/menu/list | GET | 分页查询菜品列表 |
| /meal/menu/all | GET | 查询所有启用的菜品（带类型筛选） |
| /meal/menu/{id} | GET | 根据ID查询菜品详情 |
| /meal/menu | POST | 新增菜品 |
| /meal/menu/{id} | PUT | 更新菜品 |
| /meal/menu/{id} | DELETE | 删除菜品 |

### 2.2 每周食谱模块

#### 2.2.1 功能概述
每周食谱模块用于制定和发布每周的餐饮计划，支持日历视图展示和批量添加功能。

#### 2.2.2 页面布局
```
┌─────────────────────────────────────────────────────────────┐
│  每周食谱                    [选择周▼]  [批量添加]            │
├─────────────────────────────────────────────────────────────┤
│  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐               │
│  │ 周一 │ 周二 │ 周三 │ 周四 │ 周五 │ 周六 │ 周日 │               │
│  ├─────┼─────┼─────┼─────┼─────┼─────┼─────┤               │
│  │ 1   │ 2   │ 3   │ 4   │ 5   │ 6   │ 7   │               │
│  │早餐 │早餐 │早餐 │     │     │     │     │               │
│  │午餐 │午餐 │     │     │     │     │     │               │
│  │晚餐 │     │     │     │     │     │     │               │
│  └─────┴─────┴─────┴─────┴─────┴─────┴─────┘               │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2.3 功能点说明

**1. 日历视图展示**
- 月历形式展示当月食谱安排
- 每天显示该日所有餐别（早餐/午餐/晚餐）的菜品列表
- 点击日期可查看/编辑当日食谱
- 周选择器快速切换周次

**2. 添加/编辑食谱**
- 日期选择：日期选择器
- 餐别选择：单选（早餐/午餐/晚餐）
- 菜品选择：多选下拉框，从菜品库中选择
- 已选菜品展示：标签形式展示，支持删除

**3. 批量添加食谱**
- 开始日期、结束日期选择
- 按星期几分组（周一至周日）
- 每天可设置早餐、午餐、晚餐的菜品
- 批量保存生成多条食谱记录

**4. 食谱发布**
- 草稿状态的食谱可发布
- 发布后老人端可见

#### 2.2.4 API接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| /meal/plan/list | GET | 查询食谱列表（支持日期范围） |
| /meal/plan/date | GET | 根据日期查询当日食谱 |
| /meal/plan/{id} | GET | 根据ID查询食谱详情 |
| /meal/plan | POST | 新增食谱 |
| /meal/plan/{id} | PUT | 更新食谱 |
| /meal/plan/{id}/publish | PUT | 发布食谱 |
| /meal/plan/{id} | DELETE | 删除食谱 |

老人订餐模块用于管理老人的订餐记录，支持订餐、修改、取消、配送等全流程管理。

```
├─────────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────────┤
│  编号 │ 老人姓名 │ 房间号 │ 餐别 │ 预订菜品 │ 状态 │ 操作    │
│  1    │ 张大爷   │ 101   │ 早餐 │ 小米粥.. │ 待配送│ 编辑配送取消│
│  ...  │ ...      │ ...   │ ...  │ ...      │ ...  │ ...     │
├─────────────────────────────────────────────────────────────┤
│  共 XX 条  [10▼] 条/页  < 1 2 3 ... 10 >                    │
└─────────────────────────────────────────────────────────────┘
```

#### 2.3.3 功能点说明

**1. 订餐列表展示**
- 分页展示订餐记录
- 支持按老人姓名、订餐日期、餐别、状态筛选
- 表格字段：编号、老人姓名、房间号、餐别（标签）、预订菜品、特殊要求、订餐日期、状态、操作

**2. 新增/编辑订餐**
- 选择老人：下拉选择（显示姓名+房间号）
- 订餐日期：日期选择器
- 餐别选择：单选（早餐/午餐/晚餐）
- 菜品选择：多选下拉框，根据餐别过滤可用菜品
- 特殊要求：多行文本，如"少盐、软烂、不放糖等"
- 备注：选填

**3. 订单状态管理**
- 待配送：可编辑、可配送、可取消
- 已配送：只读状态
- 已取消：只读状态

**4. 配送操作**
- 点击配送按钮，记录配送时间
- 状态变更为"已配送"

**5. 取消操作**
- 二次确认弹窗
- 状态变更为"已取消"

```
     │ 取消
```

### 2.4 送餐记录模块

#### 2.4.1 功能概述
送餐记录模块用于跟踪和管理送餐任务的执行情况，包括批量送餐、送达确认、拒收处理等功能。

**1. 统计卡片**
- 总订单数：当日该餐别所有订单
- 待送餐：状态为"待送"的订单数
- 已送餐：状态为"已送"的订单数
- 拒收：状态为"拒收"的订单数

**2. 送餐列表展示**
- 支持按日期、餐别筛选
- 表格支持多选（批量操作）

**3. 确认送达**
- 弹窗输入接收人姓名
- 记录送达时间
- 状态变更为"已送"

**4. 拒收处理**
- 弹窗确认拒收原因（可选）
- 状态变更为"拒收"

**5. 批量送餐**
- 多选订单
- 批量确认送达
- 统一记录送达时间

```javascript
const formatMenuItems = (menuItems) => {
  if (!menuItems) return '-'
  try {
    const items = JSON.parse(menuItems)
    return items.map(item => item.name || item).join('、')
  } catch {
    return menuItems
  }
}
```

## 八、异常处理

### 8.1 常见异常场景

| 场景 | 处理方式 |
|-----|---------|
| 菜品被引用 | 提示"该菜品已被食谱引用，无法删除" |
