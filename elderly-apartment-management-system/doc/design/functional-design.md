# 老年公寓管理系统 - 功能设计文档

## 1. 用户角色与权限设计

### 1.1 用户角色

本系统设计了以下几种用户角色：

| 角色 | 描述 |
|------|------|
| 系统管理员 | 拥有系统最高权限，负责系统的整体管理和维护 |
| 公寓管理员 | 负责公寓的日常运营管理，包括房间分配、人员管理等 |
| 护理人员 | 负责老年人的日常护理和健康管理工作 |
| 财务人员 | 负责公寓的财务管理，包括费用收取、报表生成等 |
| 后勤人员 | 负责公寓的后勤保障工作，包括餐饮、清洁、维修等 |
| 家属 | 可以查看老年人的基本信息、健康状况和费用信息等 |

### 1.2 权限设计

| 功能模块 | 系统管理员 | 公寓管理员 | 护理人员 | 财务人员 | 后勤人员 | 家属 |
|----------|------------|------------|----------|----------|----------|------|
| 用户管理 | 全部权限 | 部分权限 | 无 | 无 | 无 | 无 |
| 房间管理 | 全部权限 | 全部权限 | 查看 | 无 | 查看 | 查看 |
| 老年人管理 | 全部权限 | 全部权限 | 部分权限 | 查看 | 查看 | 查看 |
| 护理服务管理 | 全部权限 | 部分权限 | 全部权限 | 无 | 无 | 查看 |
| 健康管理 | 全部权限 | 部分权限 | 全部权限 | 无 | 无 | 查看 |
| 餐饮管理 | 全部权限 | 部分权限 | 查看 | 无 | 全部权限 | 查看 |
| 财务管理 | 全部权限 | 部分权限 | 无 | 全部权限 | 无 | 查看 |
| 安全管理 | 全部权限 | 部分权限 | 部分权限 | 无 | 部分权限 | 无 |
| 系统管理 | 全部权限 | 部分权限 | 无 | 无 | 无 | 无 |

## 3. 业务流程设计

### 3.1 老年人入住流程

1. 家属提交入住申请
2. 公寓管理员审核申请
3. 进行健康评估
4. 制定护理计划
5. 分配房间
6. 办理入住手续
7. 录入系统信息

### 3.2 护理服务流程

1. 制定护理计划
2. 分配护理任务
3. 护理人员执行任务
4. 记录护理情况
5. 评估护理质量
6. 调整护理计划

### 3.3 费用收取流程

1. 生成费用账单
2. 通知家属缴费
3. 收取费用
4. 记录缴费信息
5. 生成财务报表

### 3.4 健康管理流程

1. 记录健康数据
2. 分析健康状况
3. 生成健康报告
4. 提供健康建议
5. 进行健康预警

## 5. 数据流转设计

### 5.1 老年人信息流转

1. 老年人入住时，录入基本信息
2. 护理人员记录健康和护理信息
3. 财务人员记录费用信息
4. 家属可以查看相关信息
5. 系统生成统计报表

### 5.2 护理服务信息流转

1. 制定护理计划
2. 分配护理任务
3. 执行护理任务并记录
4. 评估护理质量
5. 调整护理计划

### 5.3 健康信息流转

1. 采集健康数据
2. 存储健康数据
3. 分析健康状况
4. 生成健康报告
5. 提供健康建议和预警

- 数据库连接异常
