# 老年公寓管理系统

## 项目概述

老年公寓管理系统是一款面向养老机构的信息化管理平台，采用前后端分离架构，旨在为老年公寓提供数字化、智能化的管理解决方案。系统涵盖老人信息管理、健康监测、活动管理、餐饮服务、设备报修、缴费管理等核心功能，通过移动端应用为老年人和家属提供便捷的服务体验。

系统设计充分考虑了老年人的使用习惯，界面简洁直观，操作简单易上手。移动端采用响应式设计，适配各种移动设备尺寸，支持良好的触屏操作体验。后端基于Spring Boot框架构建，提供RESTful API接口，支持Swagger文档在线查看和测试。

## 技术架构

### 后端技术栈

后端系统采用Java语言开发，基于Spring Boot 2.7.18版本构建，使用MyBatis-Plus作为ORM框架，MySQL数据库存储数据，Spring Security结合JWT实现认证授权。

在数据持久层，系统采用Spring Data JPA与MyBatis-Plus相结合的方案，MyBatis-Plus提供了便捷的CRUD操作封装，减少了大量样板代码。数据库选用MySQL 8.0版本，通过JPA的自动建表功能简化了数据库初始化过程。安全方面，系统使用Spring Security框架实现细粒度的权限控制，结合JWT（JSON Web Token）实现无状态的Token认证机制，每次请求都需要携带有效的Token才能访问受保护的API接口。

项目还集成了Swagger 3.0（通过springdoc-openapi）生成API文档，开发者可以通过访问`/swagger-ui.html`在线查看和测试所有接口。日志系统采用SLF4J配合Logback，记录详细的请求响应信息和系统运行状态。

### 前端技术栈

移动端采用Vue 3框架结合Vite构建工具开发，使用Vue Router实现单页面应用的路由管理，Pinia作为状态管理方案管理全局数据，Vant 4作为UI组件库提供丰富的移动端交互组件。

开发环境使用Vite作为构建工具，它提供了极快的冷启动速度和热模块替换（HMR）能力。Vant组件库按需引入，配合unplugin-vue-components实现自动导入，有效减少打包体积。样式方面采用Sass预处理器，通过变量文件和混合宏实现主题定制和代码复用。

Axios作为HTTP客户端库，负责与后端API进行通信，封装了请求拦截器和响应拦截器，自动处理Token注入和错误响应。Day.js用于日期时间的处理和格式化，体积轻量且API兼容Moment.js。

### 数据库设计

系统数据库包含多个核心数据表，涵盖用户管理、老人信息、健康管理、活动管理、餐饮服务、设备报修等业务领域。主要数据表包括用户表（sys_user）、老人表（elderly）、房间表（room）、健康档案表（health_archive）、健康记录表（health_record）、活动表（activity）、活动报名表（activity_signup）、餐计划表（meal_plan）、药品表（medicine）、药品发放表（medicine_distribution）、护理计划表（care_plan）、护理记录表（care_record）、排班表（nurse_schedule）等。

数据库采用InnoDB存储引擎，支持事务和外键约束，保证数据的一致性和完整性。每个表都包含必要的审计字段，如创建时间、更新时间、逻辑删除标记等。ID字段采用自增策略，由MyBatis-Plus自动处理。

## 核心功能模块

### 用户认证模块

用户认证模块是系统安全访问的基础，支持用户登录、登出、密码修改等功能。系统采用JWT（JSON Web Token）进行身份认证，登录成功后服务端生成包含用户信息的Token，客户端每次请求携带该Token进行身份验证。

用户角色分为管理员（admin）、护工（nurse）、老人（elderly）和家属（family）四种，不同角色拥有不同的权限集。管理员可以管理系统所有数据，包括用户管理、老人管理、房间管理等；护工负责日常护理工作，包括健康记录录入、药品发放确认、护理记录填写等；老人可以查看个人信息、健康档案、活动信息，进行活动报名、餐饮订餐等操作；家属可以查看老人的健康状况、活动参与情况等信息。

登录接口支持用户名密码认证，验证通过后返回JWT Token和用户基本信息。后续请求通过Authorization请求头携带Token，格式为`Bearer {token}`。Token有效期可配置，默认24小时过期。密码修改功能要求用户输入原密码和新密码进行验证。

### 老人信息管理模块

老人信息管理模块负责公寓内所有老人信息的录入、查询、修改和删除等操作。每位老人包含基本信息（姓名、性别、年龄、身份证号、联系方式等）、入住信息（入住日期、房间号、入住类型等）、健康信息（健康状态、过敏史、既往病史等）、紧急联系人信息等。

系统支持按多种条件查询老人信息，包括姓名、房间号、健康状态、入住日期等。老人信息与房间信息关联，显示老人当前入住的房间位置。健康状态字段用于标识老人的健康状况，便于护工进行分类护理。

该模块还支持老人信息的批量导入导出功能，方便与其他系统进行数据交换。照片管理功能允许上传老人头像，用于人脸识别和身份确认。紧急联系人信息用于紧急情况下的通知联系，支持设置多个紧急联系人及其关系和电话。

### 健康管理模块

健康管理模块是系统的核心功能之一，包含健康档案管理和日常健康记录两个子模块。健康档案记录老人的基础健康信息，包括身高、体重、血型、血压范围、心率范围等基准数据，以及过敏史、既往病史、当前用药等重要健康信息。

日常健康记录用于记录老人每日的健康数据，包括血压、血糖、心率、体温、体重等指标。系统支持按日期查询历史健康记录，以折线图形式展示数据变化趋势，便于观察老人健康状况的演变。异常数据自动标记，提醒护理人员关注。

健康数据录入支持批量导入功能，护工可以使用测量设备导出数据后批量录入系统。健康预警功能可以设置各项指标的正常范围，超出范围时自动生成预警记录，推送给负责的护工和管理员。健康报告功能可以生成指定时间段的健康统计报告，分析老人的整体健康状况。

### 活动管理模块

活动管理模块负责公寓内各类活动的发布、报名、管理和反馈。活动类型包括文艺活动（唱歌、舞蹈、书法、绘画等）、体育运动（太极拳、棋牌、健身等）、健康讲座、节日庆典、社交活动等。

活动发布功能包含活动名称、类型、时间、地点、简介、容纳人数等信息，支持上传活动封面图片。活动状态分为待开始、报名中、进行中、已结束、已取消等几种。发布后的活动可以随时修改信息或取消，取消时自动通知已报名的参与者。

活动报名功能允许老人查看活动列表，选择感兴趣的活动进行报名。系统显示活动的已报名人数和剩余名额，报满后自动关闭报名入口。报名成功后生成报名记录，包含报名时间、状态等信息。老人可以在报名记录中查看报名状态，或取消报名。

活动结束后可以填写活动反馈，包括参与人数、活动效果、老人评价等信息。反馈数据用于评估活动质量，为后续活动策划提供参考。活动统计功能可以按月、按类型统计活动举办次数、参与人数等数据。

### 餐饮服务模块

餐饮服务模块提供每日菜单发布、订餐管理、特殊饮食记录等功能。系统维护一周的菜单计划，包含早餐、午餐、晚餐三餐的具体菜品和价格。菜单支持按日期查看，老人可以提前查看未来几天的菜单安排。

订餐功能允许老人选择菜品进行预订，系统统计每餐的订餐数量，反馈给厨房进行备餐。订餐截止时间可以灵活设置，逾期无法预订。老人可以查看自己的订餐记录和历史订餐统计。

特殊饮食管理功能记录老人的特殊饮食需求，包括低糖餐、低盐餐、流质餐、禁食某些食物等。订餐时自动过滤不符合特殊饮食要求的菜品，确保食品安全。该模块还支持每周菜单的营养分析，计算每餐的热量、蛋白质、脂肪等营养成分含量。

### 设备报修模块

设备报修模块提供设备故障报修、进度跟踪、维修记录等功能。老人发现房间或公共区域设备出现故障时，可以通过移动端提交报修申请，描述故障现象并上传照片。

报修申请包含报修类型（水电、门窗、家具、电器等）、紧急程度（普通、紧急）、故障描述、位置信息等。提交后报修单进入待处理状态，管理员分配维修人员进行处理。老人可以实时查看报修进度，了解当前状态（待分配、处理中、已完成等）。

维修人员完成维修后填写维修记录，包括故障原因、维修措施、更换材料等信息。老人可以对维修服务进行评价，评价结果用于考核维修人员的工作质量。统计功能可以按时间、按类型分析报修数据，找出故障高发的设备和区域，为设备维护提供决策依据。

### 缴费管理模块

缴费管理模块处理老人的各项费用缴纳，包括住宿费、餐费、护理费、医药费等。费用项目可以灵活配置，支持固定费用和按量计费两种模式。

费用账单按月生成，包含各项费用的明细。老人可以查看历史账单，了解费用构成和缴纳情况。缴费功能支持线上支付（需要对接支付接口）和线下缴费记录两种方式。线下缴费由工作人员录入缴费信息，包括缴费金额、缴费方式、缴费时间等。

欠费提醒功能在账单生成后自动推送提醒消息给老人或家属。催缴功能可以批量生成催缴通知。费用统计功能可以按月、按年度统计收入情况，按费用类型分析收入构成。财务报表功能支持导出指定时间段的财务数据。

### 护理管理模块

护理管理模块包括护理计划制定和护理记录填写两个部分。护理计划根据老人的健康状况和护理需求制定，包含每日护理项目、护理频次、注意事项等。护理计划支持个性化定制，不同老人可以有不同的护理方案。

护理记录由护工在完成护理工作后填写，记录护理时间、护理内容、老人状态等信息。系统自动关联护理计划和实际执行情况，统计护理完成率。护理异常记录用于记录护理过程中发现的异常情况，如老人身体不适、摔倒等，需要及时处理并上报。

排班管理功能用于管理护工的值班安排，支持按周排班，可以设置白班、夜班、休息等班次。排班表支持调班申请和审批流程。系统自动统计护工的工作时长，支持绩效考核统计。

### 药品管理模块

药品管理模块负责公寓内药品的库存管理和发放记录。药品信息包括药品名称、规格、库存数量、有效期、存放位置等。库存不足时自动生成采购建议，提醒管理员及时补充。

药品发放功能记录每位老人的日常用药发放情况，包括发放时间、药品名称、数量、发放人、确认人等信息。发放记录与护理记录关联，形成完整的用药闭环。特殊药品（如管制药品）需要双人确认，确保用药安全。

用药提醒功能在预设的用药时间推送提醒消息给护工，确保老人按时服药。药品有效期管理功能定期检查库存药品的有效期，临近过期的药品自动预警，提示优先使用或进行报废处理。

## API接口文档

### 认证接口

认证模块提供用户登录和Token验证功能。登录接口`POST /auth/login`接收用户名和密码参数，验证成功后返回JWT Token和用户信息。Token在后续请求中通过Authorization请求头携带，格式为`Bearer {token}`。

登出接口`POST /auth/logout`使当前Token失效，客户端清除本地存储的Token和用户信息。用户信息接口`GET /auth/info`返回当前登录用户的详细信息，包括角色、权限等。修改密码接口`PUT /auth/password`允许用户修改自己的登录密码，需要提供原密码进行验证。

### 用户管理接口

用户管理接口提供用户的增删改查功能。查询接口支持按用户名、角色等条件筛选，分页返回用户列表。新增用户接口需要提供用户名、密码、姓名、角色等信息，用户名不允许重复。修改用户接口可以更新用户的基本信息和角色。删除用户接口支持单个删除和批量删除，删除操作采用逻辑删除方式。

### 老人管理接口

老人管理接口提供老人信息的完整CRUD操作。查询接口支持按姓名、房间号、健康状态等条件筛选，分页返回老人列表。新增老人接口需要填写基本信息和健康信息等字段。修改老人接口可以更新老人的各项信息，包括入住房间变更、健康状态更新等。

### 健康档案接口

健康档案接口提供老人健康档案的查询和修改功能。每位老人只有一份健康档案，包含基础健康信息。档案查询接口根据老人ID返回档案详情。档案修改接口允许更新过敏史、既往病史、当前用药等字段。

健康记录接口提供日常健康数据的增删改查。查询接口支持按日期范围查询某老人的健康记录列表。新增记录接口需要填写各项健康指标数据。数据统计接口返回指定时间段内的健康数据统计报表，包括平均值、最大值、最小值等。

### 活动管理接口

活动管理接口提供活动的发布、查询、报名等功能。活动列表接口支持按类型、时间范围、状态等条件筛选，分页返回活动列表。活动详情接口返回活动的完整信息和当前报名人数。活动发布接口需要填写活动的各项信息，发布成功后状态为"报名中"。

活动报名接口允许老人报名参加活动，需要提供活动ID和老人ID。取消报名接口允许在活动开始前取消报名。报名列表接口返回某老人或某活动的所有报名记录。活动反馈接口允许参与者在活动结束后提交反馈评价。

### 餐饮管理接口

餐饮管理接口提供菜单管理和订餐服务。菜单列表接口返回指定日期范围的餐饮计划，包含每餐的菜品明细。订餐接口允许老人选择菜品进行预订，生成订餐记录。订餐列表接口返回某老人的订餐历史。订餐统计接口返回某日期的订餐汇总数据。

### 报修管理接口

报修管理接口提供设备报修的提交和处理。报修提交接口需要填写报修类型、故障描述、位置等信息，支持上传故障照片。报修列表接口支持按状态、时间范围等条件筛选。报修详情接口返回报修的完整信息和处理进度。

报修处理接口允许管理员分配维修人员，维修人员更新处理状态。完成维修时填写维修记录。报修评价接口允许提交者对维修服务进行评分和评价。

### 缴费管理接口

缴费管理接口提供账单生成和缴费功能。账单列表接口返回老人的所有账单，支持按时间范围筛选。账单详情接口显示某账单的完整明细，包含各项费用构成。缴费接口记录缴费信息，支持线上和线下两种方式。缴费历史接口返回老人的缴费记录汇总。

## 数据库表结构

健康记录表（health_record）存储日常健康数据，主要字段包括：id、elderly_id、record_date（记录日期）、blood_pressure_high（高压）、blood_pressure_low（低压）、heart_rate（心率）、blood_glucose（血糖）、temperature（体温）、weight（体重）、remark（备注）、record_by（记录人）、create_time。

### 活动相关表

活动表（activity）存储活动信息，主要字段包括：id、name（活动名称）、type（活动类型）、description（活动描述）、activity_date（活动日期）、time_range（时间段）、location（活动地点）、max_participants（最大参与人数）、current_participants（当前报名人数）、cover_image（封面图片）、status（状态）、organizer（组织者）、create_time、update_time。

活动报名表（activity_signup）存储报名记录，主要字段包括：id、activity_id、elderly_id、signup_time（报名时间）、status（状态：已报名/已取消/已完成）、remark（备注）。活动反馈表（activity_feedback）存储活动评价，主要字段包括：id、activity_id、elderly_id、rating（评分）、comment（评价内容）、feedback_time。

餐饮计划表（meal_plan）存储每日菜单计划，主要字段包括：id、plan_date（计划日期）、meal_type（餐次：早餐/午餐/晚餐）、dish_name（菜品名称）、dish_description（菜品描述）、price（价格）、calories（热量）、create_time。

报修表（repair）存储报修信息，主要字段包括：id、elderly_id（报修人）、repair_type（报修类型）、description（故障描述）、location（位置）、images（现场图片）、priority（紧急程度）、status（状态：待分配/处理中/已完成/已评价）、repair_by（维修人）、complete_time（完成时间）、create_time、update_time。

## 快速开始

### 环境准备

后端开发需要安装JDK 11或以上版本、Maven 3.6以上版本、MySQL 8.0数据库。移动端开发需要安装Node.js 16以上版本和NPM或YARN包管理器。建议使用IDE进行开发，后端推荐IntelliJ IDEA，移动端推荐VS Code或WebStorm。

### 后端配置

修改`backend/src/main/resources/application.yml`中的数据库配置，将`spring.datasource.username`和`spring.datasource.password`修改为实际数据库用户名和密码。确保MySQL服务已启动，且数据库`elderly_apartment`已创建（如未创建，系统启动时会自动创建）。

```bash
cd backend
mvn clean install
mvn spring-boot:run
```

后端服务启动后，访问`http://localhost:8080/api/swagger-ui.html`可查看API文档。

### 移动端配置

```bash
cd mobile
npm install
npm run dev
```

移动端服务启动后，访问`http://localhost:3000`可在浏览器中预览移动端页面。移动端会自动代理后端API请求，配置在`vite.config.js`中的`server.proxy`选项。

## 项目特点

本系统在设计上充分考虑了老年公寓的实际业务需求和管理流程，具有以下特点：

系统架构采用前后端分离设计，前后端可以独立开发和部署，通过标准的RESTful API进行数据交互。这种架构便于团队协作开发，也便于后续的功能扩展和技术升级。前端采用Vue 3的Composition API组织代码，逻辑复用更加方便；后端采用Spring Boot的自动配置机制，开发效率高。

权限控制采用RBAC（基于角色的访问控制）模型，支持细粒度的权限管理。通过用户、角色、权限三级关系，可以灵活配置不同用户的访问权限。JWT Token机制实现了无状态的会话管理，便于分布式部署和水平扩展。

移动端界面采用Vant组件库，设计风格简洁大方，符合移动端用户的使用习惯。响应式布局适配各种屏幕尺寸，触屏交互优化了操作体验。集成Toast、Dialog等常用组件，提供了良好的用户反馈机制。

系统预留了多个扩展接口，支持与其他系统进行数据对接。可以对接智能硬件设备，实现健康数据的自动采集；可以对接支付平台，实现在线缴费功能；可以对接短信平台，实现消息推送功能。

## 扩展功能建议

根据实际业务需求，系统可以考虑以下扩展功能：

智能设备集成方面，可以接入智能手环、智能血压计等健康监测设备，自动采集老人的健康数据，减少人工录入的工作量，提高数据的准确性和及时性。通过物联网平台统一管理设备，实现设备状态的实时监控。

- 重点关注老人列表在首页优先展示
- 异常记录生成跟踪任务

**技术实现**：
- 录入健康数据时实时校验
- WebSocket推送提醒到在线护工
- 重点关注标识存入老人信息表

### 三、针对管理员（提升管理效率，辅助决策）

#### 1. 核心运营数据可视化看板

**核心价值**：让管理员快速掌握公寓整体运营情况，无需手动统计数据。

**功能描述**：展示核心运营数据（在住老人数量、重点关注老人数量、活动参与率、报修处理完成率、月度缴费完成情况），用简单图表展示，支持按日/周/月筛选。

**实现建议**：
- 仪表盘首页默认展示核心数据
- 数据卡片展示：入住率、床位使用率、活动参与率
- 饼图展示老人健康状态分布
- 折线图展示月度缴费趋势
- 支持数据导出

**技术实现**：
- 后端提供统计数据聚合接口
- 前端使用ECharts或Vant Charts展示图表
- 数据缓存优化访问速度

#### 2. 批量操作功能（老人、账单、活动）

**核心价值**：提升管理员批量处理事务的效率，减少重复操作。

**功能描述**：支持批量冻结/解冻老人账号、批量生成月度账单、批量下架过期活动，避免单个操作的繁琐。

**实现建议**：
- 老人列表支持多选，批量冻结/解冻
- 账单生成支持按月份、房间批量生成
- 活动列表支持批量下架、批量删除
- 批量操作前需二次确认

**技术实现**：
- 后端批量操作接口支持数组参数
- 批量操作记录日志
- 事务保证数据一致性

#### 3. 权限精细化分配功能

**核心价值**：避免权限冗余，提升系统安全性，贴合公寓实际管理架构。

**功能描述**：在原有4类核心角色基础上，新增"食堂管理员""维修人员""财务人员"细分角色，仅开放对应功能权限。

**实现建议**：
- 新增角色：食堂管理员（餐饮相关）、维修人员（报修相关）、财务人员（账单、缴费相关）
- 权限分配采用复选框界面，直观易操作
- 角色权限可复制，快速创建相似角色
- 权限变更记录审计日志

**技术实现**：
- 权限表增加模块分类
- 后端接口增加权限校验
- 前端根据权限动态显示功能入口
