# 老年公寓管理系统需求文档

## 1. 产品概述

### 1.1 项目名称
老年公寓管理系统（老年公寓）

### 1.2 产品定位
本系统是一个专为老年公寓设计的移动应用程序，旨在为老年人及其家属提供全面的生活、健康、服务管理功能。通过数字化手段提升老年公寓的服务质量，改善老年人的居住体验。

### 1.3 目标用户
- **主要用户**：老年公寓内的老年人
- **次要用户**：老年人的家属、护工、公寓管理人员

### 1.4 核心价值
- 为老年人提供便捷的生活服务入口
- 实现健康数据的实时监测与管理
- 提供丰富的公寓活动参与机会
- 简化费用管理与支付流程
- 加强紧急情况下的快速响应机制

### 1.5 产品特色
- 适老化设计：采用大字体、高对比度、简洁界面
- 全方位健康监护：涵盖生理指标监测、健康档案管理
- 智能化服务推荐：基于用户健康状况提供个性化服务建议
- 公寓互动平台：促进老年人之间的社交活动

## 2. 核心功能模块说明

### 2.1 用户认证模块
- **登录功能**：支持用户名/密码登录
- **注册功能**：支持新用户注册，包含基本信息录入
- **安全验证**：会话管理和自动登录功能
- **紧急求助**：一键紧急呼叫功能

### 2.2 首页仪表盘模块
- **欢迎信息**：个性化问候语，显示当前日期和天气
- **健康数据**：实时显示心率、血压、体温等关键生理指标
- **公寓公告**：滚动展示公寓最新通知和公告
- **快捷服务**：提供健康档案、活动中心、费用查询、今日食谱、房间服务、用药提醒、护理服务等快速入口
- **个性化推荐**：基于用户健康状况和偏好推荐相关内容
- **今日餐饮**：展示当日餐食安排和营养信息
- **今日活动**：显示当天安排的活动信息
- **健康提醒**：推送个性化的健康提醒

### 2.3 活动管理模块
- **活动列表**：展示公寓内各类活动，支持搜索和筛选
- **活动详情**：查看具体活动信息，包括时间、地点、内容等
- **报名参与**：支持在线报名参加活动
- **我的活动**：查看已报名和参与的活动记录
- **活动分类**：按类型分类展示活动（文化娱乐、体育健身、知识讲座等）

### 2.4 服务管理模块
- **健康档案**：维护个人健康记录，包括体检报告、病史等
- **费用查询**：查看账单、缴费记录和费用分析
- **房间信息**：查看房间详情和提交服务请求（维修、清洁等）
- **今日食谱**：查看每日餐食安排和营养搭配
- **药品管理**：用药提醒和药品信息管理
- **护理服务**：预约护理服务和查看护理计划
- **紧急服务**：快速联系医疗或其他紧急服务

### 2.5 健康监测模块
- **实时监测**：持续跟踪心率、血压、血糖、体温等生命体征
- **健康趋势**：展示健康数据的变化趋势图表
- **健康记录**：保存历次体检和诊疗记录
- **健康提醒**：设置并接收用药、体检等健康相关提醒
- **健康报告**：生成和查看健康评估报告
- **异常预警**：当健康数据超出正常范围时发出警报

### 2.6 财务管理模块
- **费用概览**：展示待缴费、已缴费和总费用统计
- **账单管理**：查看详细的月度账单信息
- **在线缴费**：支持多种支付方式完成缴费
- **缴费历史**：查看历史缴费记录
- **费用分析**：提供费用趋势分析和报告
- **缴费提醒**：设置账单到期提醒

### 2.7 个人中心模块
- **个人信息**：查看和编辑个人资料
- **数据概览**：显示未读通知、参与活动、健康档案等统计数据
- **我的服务**：访问个人相关的各类服务
- **消息中心**：查看系统通知和个人消息
- **家属管理**：管理家属联系信息
- **账户设置**：修改密码、隐私设置等
- **主题设置**：调整应用外观主题

### 2.8 通信与通知模块
- **通知中心**：集中管理各类系统通知
- **消息推送**：接收健康提醒、活动通知等
- **紧急联系**：快速联系紧急联系人或服务机构
- **社区公告**：查看社区发布的公告信息

## 3. 用户角色与权限

### 3.1 老年人用户（主要角色）
- **角色标识**：elderly
- **权限范围**：
  - 访问所有面向老年用户的功能模块
  - 查看和管理个人健康档案
  - 报名参加社区活动
  - 查询个人费用账单
  - 管理个人房间服务请求
  - 设置健康提醒和用药提醒
  - 访问个人护理服务
  - 使用紧急求助功能
  - 修改个人资料和设置
  - 接收个性化推荐内容

### 3.2 家属用户（次要角色）
- **角色标识**：family
- **权限范围**：
  - 查看关联老年人的健康数据
  - 接收健康状况通知
  - 协助管理老年人的活动报名
  - 代为处理费用缴纳
  - 接收紧急情况通知
  - （预留）协助管理老年人的日常事务

### 3.3 护工/护理人员（管理角色）
- **角色标识**：caregiver
- **权限范围**：
  - （预留）管理所负责老年人的健康数据
  - （预留）添加护理记录
  - （预留）发送健康提醒
  - （预留）处理服务请求
  - （预留）生成护理报告

### 3.4 社区管理员（管理角色）
- **角色标识**：admin
- **权限范围**：
  - （预留）管理社区公告发布
  - （预留）审核和管理活动
  - （预留）财务报表查看
  - （预留）用户管理
  - （预留）系统配置

### 3.4 权限设计原则
- **最小权限原则**：用户仅能访问其角色所需的最小功能集
- **数据隔离**：不同用户只能访问自己或授权访问的数据
- **操作审计**：重要操作需要记录操作日志
- **身份验证**：敏感操作需要二次验证

## 5. 交互逻辑与数据流转

### 5.1 用户交互流程

#### 5.1.1 登录流程
1. 用户打开应用，自动跳转至登录页面
2. 用户选择登录或注册标签
3. 在登录界面输入用户名和密码
4. 点击登录按钮，向服务器发送认证请求
5. 服务器验证凭据，返回用户信息和访问令牌
6. 应用存储令牌到本地缓存
7. 跳转至首页仪表盘

#### 5.1.2 首页数据加载流程
1. 进入首页时触发onLoad生命周期
2. 调用initData方法初始化所有数据
3. 并行获取以下数据：
   - 当前日期和用户信息
   - 即将到来的活动列表
   - 最新公告列表
   - 个性化推荐内容
   - 今日餐饮信息
   - 健康数据
4. 更新页面数据并渲染UI

#### 5.1.3 服务访问流程
1. 用户在首页或服务页面点击服务图标
2. 通过uni.navigateTo跳转到相应服务页面
3. 目标页面onLoad时获取相关数据
4. 渲染服务详情或操作界面
5. 用户执行相关操作（如报名活动、缴费等）

### 5.2 数据流转逻辑

#### 5.2.1 API请求封装
- 所有网络请求通过utils/request.js统一处理
- 自动添加认证令牌到请求头
- 统一处理错误响应和登录过期
- 支持Promise链式调用

#### 5.2.2 数据模型结构
```javascript
// 用户信息模型
{
  id: Number,
  realName: String,
  username: String,
  role: String, // elderly, family, admin
  roomNumber: String
}

// 健康数据模型
{
  heartRate: Number,      // 心率
  bloodPressure: String,  // 血压 "收缩压/舒张压"
  temperature: Number     // 体温
}

// 活动数据模型
{
  id: Number,
  title: String,
  startTime: String,
  endTime: String,
  location: String,
  status: Number,         // 0-未开始, 1-进行中, 2-已结束
  maxParticipants: Number,
  currentParticipants: Number
}

// 账单数据模型
{
  id: Number,
  billNo: String,
  billMonth: String,
  roomFee: String,
  mealFee: String,
  careFee: String,
  totalAmount: String,
  dueDate: String,
  status: Number         // 0-未缴费, 1-部分缴费, 2-已缴清
}
```

#### 5.2.3 数据存储策略
- **临时数据**：使用Vue组件data存储页面级临时数据
- **持久化数据**：使用uni.setStorageSync存储用户凭证和基本资料
- **缓存策略**：对不常变化的数据进行本地缓存以提高性能

### 5.3 状态管理

#### 5.3.1 页面状态
- 使用Vue的data属性管理页面内部状态
- 通过computed属性处理派生数据
- 使用watch监听数据变化并作出响应

#### 5.3.2 应用状态
- 用户登录状态：通过token存在与否判断
- 用户信息：存储在本地缓存中
- 页面跳转状态：通过uni路由API管理

### 5.4 交互反馈机制

#### 5.4.1 操作反馈
- **加载状态**：异步操作时显示加载动画
- **成功反馈**：操作成功时显示绿色成功提示
- **错误反馈**：操作失败时显示红色错误提示
- **警告反馈**：需要用户注意时显示黄色警告

#### 5.4.2 导航反馈
- **页面跳转**：使用uni导航API进行页面切换
- **返回操作**：支持物理返回键和虚拟返回按钮
- **标签页切换**：使用uni.switchTab切换底部标签页

### 5.5 异常处理流程

#### 5.5.1 网络异常
- 请求失败时显示友好的错误提示
- 自动重试机制（特定场景）
- 离线模式下显示缓存数据

#### 5.5.2 认证异常
- Token过期时自动跳转至登录页面
- 清除本地存储的用户信息
- 阻止后续需要认证的请求

#### 5.5.3 数据异常
- 数据格式错误时使用默认值
- 关键业务数据验证和清理
- 异常数据上报机制

## 6. 非功能性需求

### 6.1 性能要求

#### 6.1.1 响应时间
- **页面加载时间**：首页及其他主要页面应在3秒内完成加载
- **API响应时间**：95%的API请求应在2秒内返回响应
- **操作响应时间**：用户操作反馈应在0.5秒内显示
- **数据同步延迟**：健康数据同步延迟不超过5秒

#### 6.1.2 并发性能
- **用户并发数**：系统应支持至少1000个并发用户
- **请求处理能力**：每秒处理不少于100个API请求
- **数据同步**：支持多设备数据同步，确保数据一致性

#### 6.1.3 资源消耗
- **内存占用**：应用运行时内存占用不超过200MB
- **CPU使用率**：正常操作下CPU使用率不超过30%
- **电池消耗**：后台运行时电量消耗应低于同类应用平均水平

### 6.2 可用性要求

#### 6.2.1 系统可用性
- **系统可用性**：99.5%的系统正常运行时间
- **故障恢复**：系统故障后应在30分钟内恢复正常
- **备份策略**：每日自动备份，保留30天历史数据

#### 6.2.2 易用性
- **适老化设计**：符合老年人使用习惯，字体大小适宜
- **操作简便**：主要功能操作步骤不超过3步
- **学习成本**：新用户能够在10分钟内掌握基本操作
- **无障碍支持**：支持语音播报等辅助功能

### 6.3 安全性要求

#### 6.3.1 数据安全
- **数据传输加密**：所有网络通信必须使用HTTPS协议
- **敏感数据保护**：健康数据、个人资料等敏感信息需加密存储
- **访问控制**：实施严格的权限控制，防止未授权访问
- **数据脱敏**：在必要情况下对敏感数据进行脱敏处理

#### 6.3.2 身份认证
- **多因素认证**：支持短信验证码等多重身份验证
- **会话管理**：实现安全的会话管理，定期刷新令牌
- **登录防护**：实现登录失败次数限制和IP白名单功能

#### 6.3.3 隐私保护
- **数据最小化**：仅收集必要的用户数据
- **用户同意**：明确告知数据使用目的并获得用户同意
- **数据删除**：支持用户自主删除个人数据
- **合规性**：符合相关法律法规要求

### 6.4 兼容性要求

#### 6.4.1 设备兼容性
- **操作系统**：支持iOS 12.0及以上版本，Android 6.0及以上版本
- **屏幕尺寸**：适配主流手机和平板设备的屏幕尺寸
- **硬件要求**：支持2GB RAM及以上的设备运行

#### 6.4.2 浏览器兼容性
- **H5版本**：兼容主流移动浏览器（Chrome、Safari、UC等）
- **响应式设计**：在不同分辨率下保持良好的用户体验

#### 6.4.3 网络环境
- **网络适应性**：支持WiFi、4G、5G网络连接
- **离线功能**：在网络不稳定时提供基础离线功能

## 7. 原型图设计指导

### 7.1 设计原则
1. **突出重点信息**：首页突出健康数据和紧急求助功能
2. **简化操作路径**：常用功能放在显眼位置，减少操作步骤
3. **视觉层次清晰**：通过颜色、大小、间距区分信息重要性
4. **适老化设计**：大字体、高对比度、简洁布局

### 7.2 原型图结构
1. **首页仪表盘**：包含欢迎信息、健康数据、快捷服务等
2. **服务页面**：分类展示各类服务功能
3. **活动页面**：展示公寓活动信息
4. **健康页面**：健康数据和档案管理
5. **费用页面**：账单和缴费管理
6. **个人中心**：用户信息和设置

### 7.3 交互设计要点
1. **导航一致性**：保持底部导航栏风格统一
2. **反馈及时性**：所有用户操作都应有即时反馈
3. **错误处理**：提供清晰的错误提示和解决方案
4. **手势友好**：支持常见的滑动手势操作
