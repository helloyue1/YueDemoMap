# App端护理模块功能展示文档

## 文档概述

本文档基于养老院公寓管理系统护理模块的后端功能设计，详细定义了App端护理模块的核心功能、数据展示内容、用户交互流程及界面元素布局。本文档可直接作为绘制App端护理模块原型图的依据。

---

## 一、模块架构与层级关系

## 二、核心功能模块详细设计

### 2.1 护理工作台（首页）

#### 2.1.1 页面功能定位
护理人员每日工作的起始页面，集中展示当日待办任务、排班信息和快捷操作入口。

#### 2.1.2 界面布局

**顶部区域（固定高度：120px）**
- **位置**：页面最顶部
- **内容**：
  - 左侧：当前日期（大号字体，如"1月15日 周一"）
  - 右侧：天气图标+温度+天气状况（如"☀️ 18°C 晴"）
  - 下方：问候语（如"早上好，李护士"）

**快捷操作区（高度：100px）**
- **位置**：顶部区域下方
- **布局**：横向排列4个快捷图标按钮
- **按钮定义**：
  1. **扫一扫**：圆形图标，用于扫描老人手环/床头二维码
  2. **紧急呼叫**：红色圆形图标，带电话标识
  3. **新增记录**：蓝色圆形图标，带+标识
  4. **消息通知**：带红点角标的铃铛图标

**今日排班卡片（高度：140px）**
- **位置**：快捷操作区下方
- **布局**：
  - 左侧：班次标识（早班/中班/晚班）带颜色区分
    - 早班：橙色背景
    - 中班：蓝色背景
    - 晚班：紫色背景
  - 中间：工作时间（如"08:00 - 16:00"）
  - 右侧：负责区域（如"A区 1-3楼"）
- **底部**：当前状态标签（"进行中"/"未开始"/"已结束"）

**今日任务统计区（高度：80px）**
- **位置**：排班卡片下方
- **布局**：横向3个统计卡片
  - 待执行：数字+"待执行"标签（红色数字）
  - 执行中：数字+"执行中"标签（橙色数字）
  - 已完成：数字+"已完成"标签（绿色数字）

**待办任务列表（剩余高度，可滚动）**
- **位置**：统计区下方
- **布局**：纵向列表，每项高度约100px
- **列表项内容**：
  - 左侧：老人头像（圆形，60x60px）
  - 中间：
    - 老人姓名+房间号（如"张大爷 101室"）
    - 护理类型标签（如"晨间护理"）
    - 计划时间（如"09:00"）
  - 右侧：
    - 优先级标识（高/中/低）
    - 状态标签（待执行/执行中）
- **交互**：
  - 点击整行进入任务详情
  - 左滑显示"开始执行"快捷按钮

#### 2.1.3 数据字段

| 字段名 | 类型 | 说明 | 来源 |
|--------|------|------|------|
| currentDate | String | 当前日期 | 系统生成 |
| weather | Object | 天气信息 | 第三方API |
| nurseName | String | 护理员姓名 | 用户信息 |
| shiftType | Integer | 班次类型(1早班/2中班/3晚班) | nurse-schedule API |
| workTime | String | 工作时间范围 | nurse-schedule API |
| responsibleArea | String | 负责区域 | nurse-schedule API |
| pendingCount | Integer | 待执行数量 | care-record API |
| inProgressCount | Integer | 执行中数量 | care-record API |
| completedCount | Integer | 已完成数量 | care-record API |
| todayTasks | Array | 今日任务列表 | care-record API |

### 2.2 护理任务模块

#### 2.2.1 任务列表页

**页面功能定位**
展示护理人员的全部护理任务，支持多维度筛选和搜索。

**界面布局**

**顶部搜索栏（高度：60px，固定）**
- **位置**：页面顶部
- **内容**：
  - 搜索输入框（占位符："搜索老人姓名/房间号"）
  - 右侧筛选按钮（漏斗图标）

**筛选标签栏（高度：50px，固定）**
- **位置**：搜索栏下方
- **布局**：横向滚动标签
- **标签项**：
  - 全部（默认选中）
  - 待执行
  - 执行中
  - 已完成
  - 已取消
- **交互**：点击切换，选中项底部有下划线标识

**任务列表（剩余高度，可滚动）**
- **布局**：纵向列表，每项高度约120px
- **列表项结构**：
  ```
  ┌─────────────────────────────────────┐
  │ [头像]  张大爷        [高优先级]    │
  │         101室         [待执行标签]  │
  │                                     │
  │  晨间护理    计划: 09:00    30分钟  │
  └─────────────────────────────────────┘
  ```
- **状态颜色标识**：
  - 待执行：灰色边框
  - 执行中：橙色边框+左侧橙色竖条
  - 已完成：绿色边框+左侧绿色竖条
  - 已取消：灰色背景+删除线

**底部操作栏（高度：60px，固定，仅执行中任务显示）**
- **位置**：页面底部
- **内容**：
  - 左侧："当前正在执行X个任务"
  - 右侧："查看"按钮

**数据字段**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 任务ID |
| elderlyId | Integer | 老人ID |
| elderlyName | String | 老人姓名 |
| roomNumber | String | 房间号 |
| careType | Integer | 护理类型(1晨间/2晚间/3饮食/4康复/5用药/6其他) |
| careTypeText | String | 护理类型文本 |
| careContent | String | 护理内容摘要 |
| careDuration | Integer | 预计时长(分钟) |
| planTime | String | 计划执行时间 |
| priority | Integer | 优先级(1低/2中/3高) |
| status | Integer | 状态(1待执行/2执行中/3已完成/4已取消) |
| elderlyAvatar | String | 老人头像URL |

#### 2.2.2 任务详情页

**页面功能定位**
展示单个护理任务的完整信息，提供开始执行、查看老人档案等操作。

**界面布局**

**顶部老人信息卡片（高度：160px）**
- **位置**：页面顶部
- **内容**：
  - 左侧：老人头像（80x80px，圆形）
  - 中间：
    - 老人姓名（大号字体）
    - 房间号+楼层信息
    - 护理等级标签（如"二级护理"）
  - 右侧：电话图标按钮（点击拨打紧急联系人电话）

**任务信息区（高度：自适应）**
- **布局**：卡片式布局
- **内容项**：
  | 标签 | 值 | 样式 |
  |------|-----|------|
  | 护理类型 | 晨间护理 | 标签样式 |
  | 计划时间 | 2024-01-15 09:00 | 普通文本 |
  | 预计时长 | 30分钟 | 普通文本 |
  | 优先级 | 高 | 红色标签 |
  | 护理内容 | 协助老人完成晨间洗漱... | 多行文本 |
  | 注意事项 | 老人行动不便，需搀扶 | 黄色背景提示 |

**历史记录区（高度：200px，可折叠）**
- **标题**："该老人近期护理记录"
- **内容**：最近3条护理记录摘要
- **交互**：点击展开完整历史记录页

**底部操作栏（高度：80px，固定）**
- **布局**：根据状态显示不同按钮
  - 待执行状态：
    - 左侧："查看老人档案"按钮（白色背景）
    - 右侧："开始执行"按钮（蓝色背景，主按钮）
  - 执行中状态：
    - 左侧："继续执行"按钮
    - 右侧："完成并记录"按钮（绿色背景）
  - 已完成状态：
    - 中间："查看护理记录"按钮

**数据字段**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 任务ID |
| carePlanId | Integer | 关联护理计划ID |
| elderlyId | Integer | 老人ID |
| elderlyName | String | 老人姓名 |
| roomNumber | String | 房间号 |
| careLevel | Integer | 护理等级(1一级/2二级/3三级/4特级) |
| careType | Integer | 护理类型 |
| careContent | String | 护理内容详情 |
| notes | String | 注意事项 |
| caregiverId | Integer | 护理员ID |
| caregiverName | String | 护理员姓名 |
| planTime | String | 计划时间 |
| careDuration | Integer | 预计时长 |
| priority | Integer | 优先级 |
| status | Integer | 状态 |
| recentRecords | Array | 近期护理记录 |

#### 2.2.3 任务执行/护理记录页

**页面功能定位**
护理人员执行任务时填写护理记录的页面，支持拍照上传、时间记录等功能。

**界面布局**

**顶部进度指示器（高度：60px）**
- **位置**：页面顶部
- **内容**：
  - 步骤指示器：开始执行 → 护理记录 → 完成确认
  - 当前步骤高亮显示

**老人信息栏（高度：70px）**
- **位置**：进度指示器下方
- **内容**：老人头像+姓名+房间号（紧凑显示）

**护理记录表单（可滚动区域）**
- **表单字段布局**：

  **1. 执行时间（必填）**
  - 标签："实际执行时间"
  - 控件：日期时间选择器（默认当前时间）
  - 位置：第1行

  **2. 护理时长（必填）**
  - 标签："实际护理时长"
  - 控件：数字输入框 + "分钟"单位
  - 位置：第2行

  **3. 护理效果（必填）**
  - 标签："护理效果"
  - 控件：单选按钮组
  - 选项：良好 / 一般 / 较差
  - 位置：第3行

  **4. 老人状况（必填）**
  - 标签："老人护理前后状况"
  - 控件：多行文本输入框
  - 占位符："描述老人护理前后的身体状况、精神状态等"
  - 位置：第4行

  **5. 异常情况（选填）**
  - 标签："异常情况"
  - 控件：多行文本输入框
  - 占位符："如有异常情况请详细描述"
  - 样式：黄色边框提示
  - 位置：第5行

  **6. 拍照记录（选填）**
  - 标签："现场照片"
  - 控件：图片上传区域
  - 布局：九宫格样式，最多9张
  - 操作：点击+号调用相机/相册
  - 位置：第6行

  **7. 备注（选填）**
  - 标签："其他备注"
  - 控件：多行文本输入框
  - 位置：第7行

**底部提交栏（高度：80px，固定）**
- **位置**：页面底部
- **内容**：
  - 左侧："保存草稿"按钮（灰色）
  - 右侧："提交完成"按钮（蓝色主按钮）

**数据字段**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| careTime | DateTime | 是 | 实际护理时间 |
| careDuration | Integer | 是 | 实际护理时长(分钟) |
| careEffect | Integer | 是 | 护理效果(1良好/2一般/3较差) |
| elderlyCondition | String | 是 | 老人状况描述 |
| abnormalSituation | String | 否 | 异常情况描述 |
| imageUrls | String | 否 | 照片URL列表(逗号分隔) |
| notes | String | 否 | 备注 |
| status | Integer | 是 | 记录状态(3已完成) |

| 场景 | 提示方式 | 持续时间 |
|------|----------|----------|
